# 后续能力 Backlog

更新时间：2026-05-05

本文只记录当前仍可作为后续任务入口的能力。已完成或过期的实施计划不保留在公开文档中。

## 当前不作为缺口处理

- Auth refresh：已通过 HttpOnly refresh cookie 落地。
- 单设备在线：已落地，同一账号新登录会撤销旧 session。
- 邮箱验证码注册/登录：已落地，验证码 hash 存储、白名单、限流和 SMTP 发送已接入。
- 管理员设置基础能力：用户、角色、会话、数据库导入导出、通用运行配置已落地。
- Workspace layout / 布局快照：已从当前产品入口移除，不再暴露前端快照管理入口。
- 审计详情深链：已落地 `/audit/:logId`。
- 传输页上传/下载入口：上传和下载属于文件页职责，传输页负责列表、详情和控制。
- 远程全量文件搜索：已通过显式 search task 面板落地。
- 审计 CSV 导出：后端异步任务和前端弹窗已落地。
- 真实 SSH/SFTP 链路：已有手动 smoke runner，不进入默认 CI 门禁。

## 可选增强

### 管理员设置

- 通用配置增加更强前端校验和字段级错误展示。
- SMTP 测试邮件结果展示更详细的服务端错误分类，但不能泄露 SMTP 密钥。
- 角色权限说明继续中文化，并为自定义角色提供更清晰的权限影响解释。
- 管理员数据库导入提供更细的预览/差异确认。

### 审计与安全

- 审计页针对新增事件类型优化筛选和详情展示。
- 增加审计导出字段选择。
- 登录失败、验证码失败等安全事件增加聚合视图。
- 终端命令/输出审计仍需单独安全设计和用户显式开启，不默认记录完整 IO。

### 终端

- 连接日志继续细化错误分类和可复制诊断信息。
- 终端历史增加更强筛选、按主机/时间分组和批量清理。
- 全屏模式继续优化快捷键、焦点和移动端行为。

### 文件管理

- 文本编辑增加远端修改冲突提示。
- 大文件预览策略和编码检测。
- 更多预览类型和更明确的下载/打开策略。
- 远程搜索任务历史清理或保留策略。

### 凭据与密钥

- 批量 rotate/re-encrypt CLI 尚未落地；当前只有 versioned encryptor 和 `credential key-status`。
- 凭据导入导出继续保持密文备份，不导出明文 secret。

### 团队/RBAC

- 团队空间、组织级 RBAC、凭据共享和团队审计仍是后续方向。
- 当前管理员角色只是平台管理能力，不等同于团队资源共享模型。
- 设计背景见 `docs/architecture/team-rbac.md`，实施前需要重新复核当前代码和产品目标。

## 维护规则

- 新增 backlog 时写清楚是否需要后端 schema/API 变化。
- 已完成项不要长期留在 backlog 主体，完成后移动到“当前不作为缺口处理”或直接删除。
- 如果某项设计已经过期，从当前公开文档中移除，避免把它当作当前计划。
