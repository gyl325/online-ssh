## 1. 文档目标

明确第一版产品范围、非范围、验收边界与阶段目标，防止开发过程范围漂移。

---

## 2. 产品定位

第一版定位为：**面向个人用户的在线 SSH 与远程文件管理工具**。

核心价值：

1. 通过浏览器连接和管理多台远程主机
2. 提供接近桌面 SSH 工具的终端体验
3. 提供稳定、可恢复的远程文件传输能力
4. 提供清晰的操作日志与后续审计扩展基础

---

## 3. 第一版必做范围

### 3.1 账户与登录

- 用户注册 / 登录 / 登出
- 登录日志
- 登录设备信息展示

### 3.2 主机管理

- 新增主机
- 编辑主机
- 删除主机
- 收藏 / 搜索
- 支持密码认证
- 支持私钥认证（文本输入私钥内容）
- 保存凭据并加密存储
- host fingerprint 首次确认与变化提示

### 3.3 终端

- 浏览器终端
- 多标签连接多台主机
- 标签切换与关闭
- 基础常用命令收藏
- 复制、粘贴、resize

### 3.4 文件管理

- 远程目录浏览
- 刷新
- 搜索
- 新建文件夹
- 重命名
- 删除
- 下载
- 上传
- 基础 chmod

### 3.5 文件传输

- 本地拖拽上传
- 下载远程文件
- 任务列表
- 暂停/恢复/取消/重试
- 大文件断点续传
- 平台中转

### 3.6 审计

- 登录日志
- 连接日志
- 文件操作日志
- 文件传输日志
- 用户可查看自己的记录

### 3.7 UI

- 中文 / 英文
- 深色 / 浅色主题（可后补，但建议预留）
- 简洁现代风格

---

## 4. 第一版明确不做

1. 完整团队协作
2. 完整 RBAC
3. 会话审批流
4. 远程桌面
5. Kubernetes / Docker 专项运维
6. 浏览器原生本地目录面板
7. 默认开启的命令/输出完整审计
8. 远端到远端直传

---

## 5. 第一版延后但要预留

1. 终端标签拖拽分屏完整体系
2. 工作区布局持久化
3. 用户自定义审计级别
4. 命令审计
5. 输出审计
6. 团队空间
7. RBAC

---

## 6. 第一版关键产品定义

### 6.1 下载到本地

产品定义为：

- 用户可以从远程下载文件到本地
- 保存位置由浏览器下载流程决定
- 不承诺桌面应用式的任意目录直接写入

### 6.2 私钥登录

产品定义为：

- 第一版支持文本输入私钥内容
- 不要求必须上传本地 `.pem` 文件
- 若私钥有 passphrase，则独立录入

### 6.3 审计

产品定义为：

- 第一版先记录结构化操作事件
- 后续可由用户选择开启更细粒度终端交互记录

---

## 7. MVP 成功标准

从产品视角，第一版成功至少满足：

1. 用户可以稳定登录并管理自己的主机
2. 用户可以通过浏览器进入终端完成常见 SSH 操作
3. 用户可以上传、下载并恢复大文件传输
4. 用户可以查看自己的操作与传输记录
5. 用户不会因为安全设计缺失而暴露凭据

---

## 8. 建议产品演进顺序

### 阶段 A：打通闭环

- 登录
- 主机管理
- SSH 终端
- 远程目录浏览
- 文件上传/下载
- 凭据加密
- 基础审计

### 阶段 B：增强可用性

- 传输暂停/恢复/重试
- 大文件断点续传强化
- 常用命令
- 搜索、重命名、删除、chmod
- 任务历史

### 阶段 C：增强体验

- 更完整的工作区
- 更丰富的 UI 动效
- 更强日志筛选
- 布局保存

---

## 9. 本文结论

第一版应聚焦：**个人用户、多主机管理、浏览器终端、平台中转文件传输、断点续传、安全存储、操作审计。**
